本周主线

如果只看结论，这一周大概可以概括成下面五段。

03-08
架构换血 + 技能体系补齐

TypeScript 重构、项目技能发现、`/sk_*` 命令、Telegram 流式预览和图片输入修复，属于基础能力重建阶段。

03-09 ~ 03-10
交互观感与隐私安全开始收口

README 重写、等待态和计划流可视化、`/host` 能力增强、Claude 1m 选择修复，以及 Telegram 日志脱敏，明显从“功能堆叠”转向“可用性打磨”。

03-11 ~ 03-12
从 Telegram 工具，长成多渠道产品

工作目录持久化、Feishu 接入、Dashboard 上线、Gemini 驱动加入，产品形态从单 Bot 扩展到统一 IM 桥接平台。

03-13 ~ 03-14
MCP、品牌切换、Feishu 稳定性爆发式推进

MCP 实时回传文件、项目更名为 Pikiclaw、截图工具拆模块、`.pikiclaw/skills` 收敛，以及 Feishu/CardKit/Gemini 权限/Stop 流程的大量补丁。

03-15
开始做 GUI 与扩展生态

Codex human-loop、桌面自动化、Playwright/Appium 扩展页、Dashboard 状态细节和品牌图标，都在说明产品正在进入“可扩展操作层”阶段。

代表版本

没有把 52 个 patch 全铺开，而是保留最能代表这一周方向变化的节点。

v0.2.13 2026-03-08

技能发现与 Telegram 体验同步升级

项目技能从 `.claude/commands` / `.claude/skills` 自动发现，新增 `/sk_*` 直接触发；同时改进 Codex 流式预览、任务状态展示和本地图片输入，是周初最明显的基础体验升级。

v0.2.19 ~ v0.2.24 2026-03-09 / 03-10

计划流、主机信息与日志脱敏

这段发布把 Telegram 计划预览、token/ctx footer、`/host` 细节、Claude 1m 选型修复以及日志中敏感内容的清理串在一起，说明稳定性和可观测性已经被提到较高优先级。

v0.2.28 2026-03-12

Feishu + Dashboard + Gemini 一次性到位

这是本周最像“平台级跃迁”的版本。Feishu Bot、浏览器 Dashboard、驱动化 agent 架构和 Gemini 支持一起落地，Pikiclaw 从单一 Telegram 机器人升级成多入口的本地代理桥。

v0.2.34 ~ v0.2.36 2026-03-13

MCP 实时回传 + Pikiclaw 品牌切换

MCP bridge 让文件可在流中直接回到 IM，会话侧工具也开始模块化；随后仓库、CLI、文档、技能路径统一更名为 `pikiclaw`，并补上截图工具与主机环境识别，产品边界更完整了。

v0.2.42 ~ v0.2.49 2026-03-14

Feishu 稳定性与控制能力狂补

一天 14 个版本，基本都围绕 Feishu/CardKit、Gemini 权限、`im_send_file` 路径解析、统一启动提示、流式卡片回退，以及跨 Telegram/Feishu 的 `stop` 中断能力展开，是本周最集中的修补窗口。

v0.2.50 ~ v0.2.53 2026-03-15

进入桌面自动化和扩展管理阶段

Codex human-loop、Appium Mac2 桌面工具、Playwright MCP 扩展配置、Dashboard 扩展页、品牌图标与状态刷新接连落地。这一组版本最能体现产品开始往“可操作 GUI + 可安装扩展”走。
